# tarka вҖ” Security / Hardening Audit (2026-06-22, v0.8.0)

> Satisfies the v1.0 criterion *"Security audit pass (`docs/audit/YYYY-MM-DD-audit.md`)"*.

## Scope & threat model

tarka is a sovereign Cyrius **reinforcement-learning + reasoning reference** вҖ” numerical
compute (policy/value nets, reward models, beam search) over rosnet `f64` tensors, tyche
PRNG, and the akshara tokenizer. It is **not** a network service or a parser of untrusted
input at runtime: the demo drives a hardcoded corpus, and akshara's file/stdin loader paths
are deliberately stubbed unreached. The real attack surface is therefore **memory safety and
robustness**, not injection / authn / web. Cyrius performs **no bounds checking** вҖ” an
out-of-range `store64` is silent heap corruption, not a trap вҖ” so the audit focused on raw
pointer indexing, allocation sizing, integer/index arithmetic, `f64` divide-by-zero / NaN,
unchecked resource caps, and RNG determinism.

## Methodology

A 6-dimension audit workflow (`tarka-08-hardening-audit`): each dimension **found** issues in
the real source, then a separate adversary **re-derived reachability** for every finding from
the actual call graph (default: downgrade unless a concrete trigger exists). Dimensions:
beam-buffer-bounds В· allocation-sizing В· index/integer-arithmetic В· numeric-robustness В·
caps/API-misuse В· input-surface/RNG. 12 agents, ~1.0M tokens.

## Posture

**tarka is memory-safe as called today.** The audit found **zero reachable bugs** вҖ” every
call site in the demo, the grad-check suite, and the public API stays within all buffer
bounds and never divides by zero. The findings were **hardening gaps**: unchecked
*preconditions* on public functions that a future caller could trip into silent heap
corruption (or a hang / NaN). 0.8.0 closes the memory-safety-relevant gaps with fail-loud
precondition guards and accepts the remaining low-risk NaN-on-nonsensical-input items with
rationale.

## Findings & resolution

Severity reflects worst-case *if the precondition is violated* (none are reachable from
current callers). All fixes are **additive** вҖ” the guards are no-ops when preconditions hold,
so the 24 grad-checks and all demo gates are byte-identical.

### The guard mechanism

stdlib `assert` only *prints* "FAIL" and continues вҖ” it does not stop, so it cannot prevent
a subsequent corrupting write. 0.8.0 adds a self-contained fail-loud `guard(cond, msg)`
(`rl.cyr`) that prints `tarka: precondition violated: <msg>` to stderr and `SYS_EXIT(1)` on a
false condition. Verified end-to-end: it aborts on violation and is a clean no-op when
satisfied. 13 guard sites across the public surface.

## Coverage вҖ” checked and cleared

- **Allocation sizing** вҖ” every `t_alloc` (policy/critic/reward params + grads + Adam moments,
  `R_*` rollout buffers, `GR_*` group buffers, beam buffers) covers the maximum index its
  consumers reach for the dimensions set by `pol_init`/`*_init`.
- **Index/integer arithmetic** вҖ” `i*8` offsets and size products (`g_V*g_D`, `BMAX*BMAX`,
  `B*K`, `GMAX*g_T`) cannot overflow i64 at any realistic dimension; the `-1` sentinels are
  only read on paths the new guards now exclude.
- **RNG** вҖ” `rng_u64() & 0x7FFFвҖҰFFF` masks the sign bit before `% g_V` (non-negative modulo);
  seeding is deterministic; the per-prompt reseed offset does not overflow at demo scale вҶ’
  the grad-check suite and gates are reproducible.
- **Input surface** вҖ” akshara's loader/stream paths are confirmed unreachable from tarka and
  their stubs are side-effect-free (`return -1`, no buffer writes); `corpus_set` reads a
  NUL-terminated hardcoded string within bounds.

## Toolchain

Audited against, and pinned to, **cyrius 6.2.37** (`cyrius.cyml [package].cyrius`, bumped
from 6.2.36 this cycle). Build clean, 24/24 grad-checks, all gates pass on the pinned

## Conclusion

No reachable vulnerabilities. The memory-safety-relevant preconditions are now guarded
fail-loud; the residual items are NaN-on-nonsensical-input with no path from the reference's
own usage. **Audit: PASS.**
